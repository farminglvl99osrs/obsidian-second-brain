# Obsidian Second Brain for Claude

A git-backed Obsidian vault that Claude reads from and writes to via MCP, creating a persistent, interconnected knowledge graph that grows as you work.

## How It Works

```
Claude Code ──► MCP Server ──► Obsidian REST API ──► Vault (markdown + wikilinks)
```

- **Session start:** Claude pulls latest notes and searches for context on your current work
- **During work:** Claude appends discoveries (API relationships, architecture decisions, operational quirks) to the relevant notes
- **Session end:** Say "wrap it up" and Claude creates a session summary linking everything touched

## Vault Structure

```
services/           One note per repo/service, with tech stack, endpoints, relationships
architecture/       Cross-cutting design decisions and ADRs
operations/         Deployment patterns, infra, runbooks, environment setup
relationships/      Service maps connecting everything
sessions/<name>/    Per-user session summaries (episodic memory)
templates/          Note templates for consistency
```

Notes use **YAML frontmatter** for structured filtering and **[[wikilinks]]** for graph connectivity. Open the vault in Obsidian to see the graph view.

## Setup (5 minutes)

### Prerequisites

- [Obsidian](https://obsidian.md/) installed
- [Node.js](https://nodejs.org/) 18+ (for the MCP server)
- [Claude Code](https://claude.ai/code) CLI

### Steps

1. **Clone this repo** where you want your vault:

   ```bash
   git clone https://github.com/farminglvl99osrs/obsidian-second-brain.git ~/Documents/my-vault
   cd ~/Documents/my-vault
   ```

2. **Open as an Obsidian vault:**
   - Open Obsidian > "Open folder as vault" > select the cloned directory

3. **Install the REST API plugin:**
   - Settings > Community plugins > Enable community plugins
   - Browse > Search "Local REST API" > Install (by Adam Coddington) > Enable
   - Copy the API key from the plugin settings

4. **Run the setup script:**

   ```bash
   ./setup.sh
   ```

   This will:
   - Verify prerequisites (Node.js, npx)
   - Configure the Obsidian MCP server in `~/.claude/mcp.json`
   - Create your personal session directory

5. **Add behavioral rules** to your Claude auto-memory (the setup script prints the exact text)

6. **Restart Claude Code** to load the MCP server

## Team Usage

This vault is designed for team sharing via git:

- Each person clones the repo and opens it as their own Obsidian vault
- Each person runs `./setup.sh` to configure their local MCP connection
- Session directories are namespaced per user (`sessions/dylan/`, `sessions/alex/`)
- Architecture decisions can go through PR review
- `.obsidian/` is gitignored (personal settings stay local)

## Note Templates

### Service Note (`templates/service.md`)

For documenting repos and services: overview, tech stack, endpoints, relationships, operational notes. Frontmatter includes `type`, `status`, `repo`, `owner`, `tags`, `last_updated`.

### Session Summary (`templates/session.md`)

Auto-generated by Claude on "wrap it up". Captures context, learnings, decisions, and open questions with links to services and tickets.

### Architecture Decision (`templates/architecture-decision.md`)

For cross-cutting decisions: the decision, the context (why), and the consequences (impact on services).

## Claude's Behavioral Rules

These go in your Claude auto-memory (`MEMORY.md`):

```markdown
## Obsidian Second Brain (Vault: <path-to-vault>)
- **Start:** Run `git pull` in the vault, then use Obsidian `search` tool for context.
- **During:** When discovering new API relationships, architecture decisions, or
  operational quirks, use `append_content` or `patch_content` to update the relevant
  note. Always update `last_updated` frontmatter.
- **End:** When I say "wrap it up", create a session summary in
  `sessions/<your-name>/YYYY-MM-DD-<topic>.md`, then git add, commit, and push.
```

## Security

- Obsidian REST API is bound to `127.0.0.1` only (localhost)
- Bearer token stored in environment variable, never committed
- No external network access
- `.obsidian/` gitignored (personal settings stay local)
